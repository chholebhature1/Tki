Version: 1.0

# 1. Purpose

Defines RLS policies for every table in TalkIndia.

RLS is mandatory. Every table must have `ENABLE ROW LEVEL SECURITY`.

# 2. Helper Functions

```sql
-- Get current user's role
CREATE FUNCTION public.get_user_role()
RETURNS text AS $$
  SELECT r.name FROM profiles p
  JOIN roles r ON r.id = p.role_id
  WHERE p.id = auth.uid()
$$ LANGUAGE sql STABLE SECURITY DEFINER;

-- Check if current user is admin
CREATE FUNCTION public.is_admin()
RETURNS boolean AS $$
  SELECT public.get_user_role() = 'admin'
$$ LANGUAGE sql STABLE SECURITY DEFINER;

-- Get current user's therapist_profile_id
CREATE FUNCTION public.get_therapist_profile_id()
RETURNS uuid AS $$
  SELECT id FROM therapist_profiles
  WHERE profile_id = auth.uid()
$$ LANGUAGE sql STABLE SECURITY DEFINER;
```

# 3. Policies by Table

## profiles

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT | Owner | `id = auth.uid()` |
| UPDATE | Owner | `id = auth.uid()` |
| INSERT | Owner | `id = auth.uid()` |
| SELECT, UPDATE | Admin | `is_admin()` |

## roles

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT | Everyone | `true` (read-only lookup) |

## therapist_profiles

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT | Public | `verification_status = 'approved'` |
| SELECT | Owner | `profile_id = auth.uid()` |
| UPDATE | Owner | `profile_id = auth.uid()` |
| INSERT | Owner | `profile_id = auth.uid()` |
| ALL | Admin | `is_admin()` |

## therapist_verifications

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT, INSERT | Owner | `therapist_profile_id = get_therapist_profile_id()` |
| UPDATE | Owner | Only when status is `draft` or `rejected` |
| ALL | Admin | `is_admin()` |

## specializations, languages

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT | Everyone | `true` (read-only lookups) |

## therapist_specializations, therapist_languages

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT | Everyone | `true` |
| INSERT, DELETE | Owner | `therapist_profile_id = get_therapist_profile_id()` |

## therapist_availability

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT | Everyone | `is_active = true` (needed for slot generation) |
| ALL | Owner | `therapist_profile_id = get_therapist_profile_id()` |

## therapist_blocked_periods

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT | Everyone | `true` (needed for slot generation) |
| ALL | Owner | `therapist_profile_id = get_therapist_profile_id()` |

## slot_locks

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT | Everyone | `true` (needed to check availability) |
| INSERT, DELETE | Service role only | Server-side booking logic |

## appointments

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT | Patient | `patient_profile_id = auth.uid()` |
| SELECT | Therapist | `therapist_profile_id = get_therapist_profile_id()` |
| SELECT, UPDATE | Admin | `is_admin()` |
| INSERT, UPDATE | Service role | Server-side booking/payment logic |

Note: Appointment creation and status updates happen server-side via service_role. Patients and therapists have read-only access.

## appointment_status_history, appointment_reschedules, appointment_cancellations

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT | Participant | Appointment belongs to user |
| SELECT | Admin | `is_admin()` |
| INSERT | Service role only | Server-side |

## reviews

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT | Everyone | `true` (public on therapist profiles) |
| INSERT | Patient | `patient_profile_id = auth.uid()` |
| UPDATE | Patient | Own review AND `created_at > now() - interval '24 hours'` |
| ALL | Admin | `is_admin()` |

## payments, payment_attempts

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT | Patient | `patient_profile_id = auth.uid()` |
| SELECT | Therapist | `therapist_profile_id = get_therapist_profile_id()` (payments only) |
| ALL | Admin | `is_admin()` |
| INSERT, UPDATE | Service role only | Server-side payment verification |

## invoices

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT | Patient | Payment belongs to patient |
| SELECT | Admin | `is_admin()` |
| INSERT | Service role only | Generated after payment |

## refunds

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT | Patient | Payment belongs to patient |
| ALL | Admin | `is_admin()` |

## video_sessions

| Action | Who | Condition |
|--------|-----|-----------|
| SELECT | Patient | Appointment belongs to patient |
| SELECT | Therapist | Appointment belongs to therapist |
| SELECT | Admin | `is_admin()` |
| INSERT, UPDATE | Service role only | Server-side |

Note: Host URL should not be exposed to patients. Use a database view or select specific columns in the repository.

# 4. Storage Buckets

| Bucket | Read | Write | Max Size |
|--------|------|-------|----------|
| `avatars` | Public | Owner (`avatars/{user_id}/*`) | 5MB |
| `verification-documents` | Owner + Admin | Owner | 10MB |
| `blog-images` | Public | Admin | 5MB |

Allowed MIME types: `image/jpeg`, `image/png`, `image/webp`, `application/pdf`

---

# 5. Key Rules

1. Enable RLS on every table immediately after creation.
2. Create helper functions before policies.
3. Use `service_role` for all server-side mutations on sensitive tables (appointments, payments, invoices).
4. Never expose `service_role` key to the client.
5. Test each policy by querying as each role.
